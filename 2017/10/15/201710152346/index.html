Why I learn Kotlin?

Because using Kotlin can help developers achieve their goals with less code and fewer ;.

Kotlin 是静态类型

每一个表达式的类型在编译时确定，并会进行编译时检查，为什么不用动态类型呢？动态类型(dynamically typed)是在运行时才确定类型，变量可以是任意类型

静态类型的好处

  • 性能更好，不需要在运行时动态的指定需要调用的方法，使得方法调用更快
  • 可靠性好，编译器会在编译时检查类型，因此会避免很多运行时异常
  • 可维护性，对不熟悉的代码更容易理解，因为可以看到对象使用的类型
  • 工具支持，静态类型可以支持重构，自动补全等IDE特性

Type Inference
定义变量时，不必申明变量的数据类型，只需要给定一个初始值，编译器会根据初始值的类型，自动确定变量的类型。

Kotlin 相对于 Java 的类型系统，新增的特性有 nullable types 和 function types

函数式编程

传统的 java 编程思想是面向对象，函数式编程是一种新的思想，它的核心概念是：

  • 函数是头等公民，可以把函数保存到变量中，可以把函数作为参数传递，也可以返回一个函数。
  • 不可变性，对象一旦创建，就不可再被改变。
  • 没有副作用，同一个输入返回的输出永远都相同

函数式编程的好处

  • 简洁，可以通过将公共基础部分抽象出来，以函数作为参数而增加灵活性，避免重复代码
  • 多线程安全，使用不可变的数据结构和函数式编程，则不用担心多线程安全问题
  • 易于测试，对于某一段函数的测试不用考虑太多的环境配置

Kotlin 对函数式编程的支持

  • 函数类型，可以接受一个函数作为参数，或者返回一个函数
  • lambda 表达式，支持lambda表达式
  • data class，一种简洁的定义类的方式，用于定义不可变的数据型类，类似javabean
  • 丰富的支持函数式编程的API

函数

表达式(expression)和语句(statements)的区别

表达式具有值，可以被求值，所以一般来说用在等号右边，而语句不一定有值，只是被执行，在 Kotlin 中，if 是表达式，因此可以在等号右边直接使用 if(这在 java 中是不行的)

变量

变量创建的时候可以省略类型申明，编译器会自动根据初始化的赋值来确定变量的类型

val 关键字申明的变量是不可变的，类似 java 中的 final
var 关键字申明的变量是可变的

但这里说的可变与不可变，都是指引用，引用所指向的内容是可以变的

Kotlin 推荐使用 val 关键字申明变量，避免副作用(side effects)

字符串

通过在字符串中使用 $ 加上变量名，来引用变量到字符串中，底层是通过 StringBuilder 实现的
可以通过 ${} 来引用一段表达式或者引用数组中的某一个内容

数据类

类似 java 中的 javaBean 类，在 Kotlin 中，这样的类不再需要手动写 getter 和 setter 方法，只需要定义即可，编译器会自动为其赋予相关方法，其中通过 val 定义的变量只具有 getter 方法，而通过 var 定义的变量，具有 getter 和 setter 方法。

源文件布局

在 Kotlin 中，一个 .kt 文件可以包含多个类

when 语法

when 替换 java 中的 switch，但 when 比 switch 更强大,switch 中每一个条件都需要 break 来结束条件，否则会继续走到下一个条件，而 when 不会这样子，when 中如果一个条件匹配了，则执行完这个条件后会自动结束。如果想要匹配多个条件，可以用逗号隔开。when 的条件可以是任何对象(相比switch强大了很多)

尽量不要在 when 中创建额外的对象

smart cast

在 Kotlin 中，执行了 is 判断后，可以不再使用强制类型转换，而在 java 中，即使用 instanceof 判断后，仍然需要强制类型转换，Kotlin中强制类型转换的工作交由编译器去完成，smark cast 只在 is 条件判断后不再变化的情况下起作用

Kotlin 中没有三目运算符

Kotlin 中的 if 和 Java 中的 if 的异同
不同： Kotlin 中的 if 是表达式，具有返回值，而 java 中的 if 是语句，不能直接写在等号右边。
相同：如果括号内的语句只有一行，则可以省略掉括号

The rule “the last expression in a block is the result” holds in all cases where a block can be used and a result is expected

循环

while 和 for 循环

Kotlin 中的 while 循环和 Java 中相同，for循环则只有 java 中的 for-each 形式的循环，也就是说 for 循环不再有 for(int i=0;i<n;i++) 这样的形式了

for 循环
通过 .. 可以构造一个序列，比如数字序列 for(i in 1..100) 表示从 1 到 100 的遍历；字符序列 for(c in 'a'..'z') 表示从 a 到 z 的遍历，还有两个额外的操作可以使遍历更灵活：downTo 和 step,其中 downTo 可以用作倒序遍历，step 用作遍历间隔，例如 for(i in 100 downTo 1 step 2) 表示从 100 到 1 的遍历，间隔为 2

注: .. 语法中的范围都是闭区间，比如 1..100 表示从 1开始到 100，其中 100 是最后一个循环到的数字

Map 新操作

在 Java 中，向 Map 中新增数据的操作有 put,在 Kotlin 中，有了更加简洁的操作，对于数据输入，不仅仅可以使用 put，还能直接用等号赋值，例如 map[a] = 1 表示新增数据，key 是 a，value 是 1;

in 操作

在 for 循环中可以使用 in 来遍历，除此之外，in 还能用作条件表达式，比如判断一个数据是否在某个列表中: if(a in b..c) 这种操作的底层实现等价于if b<=a && a<=c，对于字符串的比较，是按照字母顺序比较大小的，例如 println("Kotlin" in "Java".."Scala") 由于它等价于 println("Java"<="Kotlin" && "Kotlin"<="Scala")，所以在进行字符串比较时，按照字母大小比较，结果就是 true

init() 方法和类的构造方法的区别

在 java 中，初始化工作一般放在类的构造方法里执行，而在 kotlin 中新增了 init() 方法，用于执行类的初始化工作，但同时也保留了 java 写法的构造方法，关于这两种方式的区别，
简单点说就是，在存在多个构造方法的时候，不管调用哪个构造方法，init() 都会调用，这样就避免了选择不同构造方法执行不同初始化的问题，因此建议使用 init()来做初始化工作，参考：Constructor initialization vs Init block Kotlin

init() 方法中不能 return

所以如果有需要 return 的时候，可以用 ?.let{}

空安全

Kotlin 的类型系统致力于排除空指针异常(NullPointerException)，在 Kotlin 编写的代码中，能够出现空指针异常的情况只有

  • 直接调用 throw NullPointerException()
  • 使用 !! 操作一个空类型
  • 外部的 java 代码造成空指针
  • 变量未初始化调用
