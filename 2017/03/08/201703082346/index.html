    本文记录 Java 中一些基础的概念与相关语法

    访问权限

    • 私有访问权限(private)：同一类中可访问

    • 默认访问权限(无修饰符)：同一类中，同一包中可访问

    • 受保护的访问权限(protected)：同一类中，同一包中，子类中可访问

    • 公开访问权限(public)：同一类中，同一包中，子类中，其他包中可访问

    构造器

    • 默认的，一个类是提供了无参构造器，但是一旦编写了自定义的构造器，系统就不会提供默认的构造器

    • 不论是否显性的使用 super 调用父类构造器，子类构造器总会调用父类构造器一次，也就是说，当调用子类构造器来初始化子类对象时，父类构造器总会在子类构造器之前调用

    • 如果将父类的所有构造器都修饰为private，则表示他不能被继承，没有类能够继承它

    • 初始化块总是在构造器之前执行，不能接受参数，不能被调用

    继承

    • 如果父类某方法的访问权限是 private，则子类不能重写这个方法，即使写了一个同名的方法，那也是子类自己的方法，并不是重写父类的方法

    • 创建子类对象时，系统不仅会为子类对象中的变量分配内存，也会为其父类中的变量分配内存

    创建对象的过程

    系统首先加载类（加载类的过程是先执行初始化块，再执行构造器，如果有静态初始化块，则先从顶层到底层逐步执行静态初始化块，随后才开始从顶层到底层的执行初始化块和构造器），上朔到 Object 类，先加载 Object 类，然后以次向下加载类，最后才加载该类，加载完类后就可以使用这个类创建该类的对象，为该对象的所有实例变量分配内存空间，接着开始对这些实例变量执行初始化，顺序是：先执行初始化块或声明实例变量时指定的初始值，然后再执行构造器里指定的值

    static 修饰符

    • static 变量：静态变量在内存中只有一份，JVM 只为静态分配一次内存，在加载类的过程中完成静态变量的内存分配，用类名直接访问，也可以通过对象来访问，而没有用 static 修饰的变量叫做实例变量，每创建一个类的实例，就会为实例变量分配一次内存，实例变量可以在内存中有多个(每个实例持有一个)，互不影响

    • static 方法：静态方法可以直接通过类名调用，静态方法中不能用 this 和 super 关键字，不能直接访问所属类的实例变量和实例方法，只能访问所属类的静态成员变量和成员方法。因为实例成员与特定的对象关联。

    • static 代码块：静态代码块，是在类中独立于类成员的 static 语句块，可以有多个，位置可以随便放，它不在任何的方法体内，JVM 加载类时会执行这些静态的代码块，如果 static 代码块有多个，JVM 将按照它们在类中出现的先后顺序依次执行它们，每个代码块只会被执行一次。

    • static 类：通常一个普通类不允许声明为静态的，只有一个内部类才可以。这时这个声明为静态的内部类可以直接作为一个普通类来使用，而不需先实例化它的外部类

    final 修饰符

    • final 修饰的成员变量，要么在定义时赋初始值，要么在初始化块或构造器两者中的一个中赋初始值

    • final 修饰的局部变量，可以在定义时赋初始值，也可以在使用时赋初始值，但是只能有一次赋值的过程

    • final 修饰的方法不可以被重写，final 修饰的类不可以有子类

    • final 修饰基本数据类型和引用数据类型的区别：final修饰的基本数据类型不能被重新赋值，因此基本数据类型是不能被改变的，但是引用数据类型，它保存的仅仅是一个地址，虽然地址同样也不能变，但是地址所指向的数据是可能会变的

    常量池

    java 中使用常量池来管理曾经用过的字符串直接量，起到缓存的作用，例如执行 String a="java"; 则常量池中就会缓存一个字符串 “java”，后续再执行 String b="java"; 时就会让 b 指向常量池中的 “java”，于是 a == b 为 true，因为它们指向的同一个地址

    Lambda表达式

    主要作用是代替匿名内部类的繁琐语法。由三部分组成：1.形参列表 ()，2.箭头 -> ，3.代码块 {}

    Runtime

    Runtime 类代表 java 运行时环境，可以访问 JVM 的相关信息，每个 java 程序都有一个与之对应的 Runtime 实例

    String，StringBuffer 和 StringBuilder

    • String 类是不可变的，一旦创建一个 String 类的对象，则不可修改其中的字符序列

    • StringBuffer 类的对象中的字符串是可以修改的，通过 append(),inster(),reverse() 等方法可以实现字符串的修改，通过 toSting() 方法可以将其转换成 String 对象

    • StringBuilder 类和 StringBuffer 基本相同，其对象中的字符串也是可以修改的，但是 StringBuffer 是线程安全的，而 StringBuilder 不是线程安全的，因此性能略高

    因此，在通常情况下，要创建一个内容可变的字符串对象，优先考虑使用 StringBuilder 类

    ThreadLocalRandom

    是 Random 的升级版，用于生成一个伪随机数，在并发环境下使用可以减少多线程资源竞争，保证系统具有线程安全

    Calendar

    • add()方法与 roll() 方法的区别：当修改的内容超出允许范围，会发生进位，上一级字段会增大，而使用roll()进位时，上一级不会增大

    延迟修改：如果调用了 set() 方法修改一个字段之后，并没有立即计算出修改后的值，而是在调用 getTime() 之前计算，所以，如果在调用 getTime() 之前有多次 set()，则会将每次的设置放在一起计算，所以这样会造成一个问题，比如当前 set(2015,7,31)，通过 set() 方法设置月 set(Calendar.MONTH,8) 此时如果调用 getTime() 得到的日期为”2015.10.01”，因为9月31号自动进位到10月1号，而如果此时不调 getTime() 就直接继续设置：set(Calendar.DATE,5) 现在将月设置为9，日设置为5，则 getTime() 得到的结果为”2015.09.05”，因为这里在 getTime() 之前有两次设置，但是都是在 getTime() 之前一起计算的，于是月设置为9，日设置为5，得到的结果不需要进位。

    集合类

    • set：无序，不可重复

    • List：有序，可重复

    • Map：键值对映射关系,HashMap,线程不安全，key,Value允许为空；HashTable,线程安全，key,Value不允许为空

    • Queue：队列的集合

    泛型

    • 不管泛型类型的实际类型参数是什么，它们运行时总有同样的类，也就是说不管为泛型的形参传入哪一种类型参数，他们依然被当作同一个类处理：

    • 假设 Foo 是 Bar 的子类，则，Foo[] 依然是 Bar[] 的子类，但是 List 不是 List 的子类

    Volatile

    Volatile 修饰符解决的问题是 JVM 在-server模式下(注意普通运行模式下没有此问题),线程优先取用自己的线程私有 stack 中的变量值, 而不是公共堆中的值,造成变量值老旧的问题.

    换句话说, Volatile 强制要求了所有线程在使用变量的时候要去公共内存堆中获取值, 不可以偷懒使用自己的.
    Volatile 绝对不保证原子性, 原子性只能用 Synchronized 同步修饰符实现.

    线程

    • Thread 类中的 start() 和 run() 方法有什么区别？
      start() 方法被用来启动新创建的线程，而且 start()内部调用了 run() 方法，这和直接调用 run() 方法的效果不一样。直接调用 run() 方法的时候，只会是在原来的线程中调用，没有新的线程启动，start()` 方法才会启动新线程。

    • Java 中 Runnable 和 Callable 有什么不同？
      Runnable 和 Callable 都代表那些要在不同的线程中执行的任务，它们的主要区别是 Callable 的 call() 方法可以返回值和抛出异常，而 Runnable 的 run() 方法没有这些功能。Callable 可以返回装载有计算结果的 Future 对象
