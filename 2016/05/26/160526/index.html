349.Intersection of Two Arrays

Description:

Given two arrays, write a function to compute their intersection.

Example:

Given nums1 = [1, 2, 2, 1], nums2 = [2, 2], return [2].

Note:

  • Each element in the result must be unique.

  • The result can be in any order.

解决：

寻找两个给定数组的交集，并且不能有重复元素。这里想到有两种方法解决

  • 第一种方法：
    两层 for 循环遍历，寻找在 nums1 中存在，并且在 nums2 中存在的元素，最后再遍历一遍结果数组，将重复的元素删除，这种方式的时间复杂度至少是 O(m*n)

  • 第二种方法：
    使用 HashSet 这种数据结构来存放数据，首先将 nums1 的数据存放在 HashSet 中，然后对 nums2 进行遍历，如果 HashSet 中存在 nums2 中当前遍历的元素，则将这个元素加入结果集合中，最后输入结果集合，这种方式的时间复杂度至少为 O(m+n)

344.Reverse String

Description:

Write a function that takes a string as input and returns the string reversed.

Example:

Given s = “hello”, return “olleh”.

解决：

这道题的解决办法有很多，在 Java 中甚至可以用 StringBuilder 自带的 reverse() 方法来完成
对字符串的反转操作，这里给出两种解决：

  • 第一种方法：
    使用 for 循环从输入的字符串从尾到头逆向遍历，将结果依次存入新的字符串中，返回这个新的字符串，这种方式的时间复杂度至少是 O(n)

  • 第二种方法：
    从数组头和尾向中间遍历，将第一个与最后一个元素交换，第二个与倒数第二个交换，依此直到进行到数组中间，这种方式中，假设数组长度为 n，则循环需要的次数为 n/2，时间复杂度为 O(n/2)，显然时间上要优于第一种方式，但是在将前后两个元素交换的过程中，需要定义两个临时变量，来暂时存放当前要交换的两个字符，所以这种方式需要额外的空间复杂度为 S(n)，Java 中的 StringBuilder 类中的 reverse() 方法就是使用这种方式。

345.Reverse Vowels of a String

Description:

Write a function that takes a string as input and reverse only the vowels of a string.

Example 1:

Given s = “hello”, return “holle”.

Example 2:

Given s = “leetcode”, return “leotcede”.

解决：

将一个字符串中的所有元音字母反转，这里说两种很容易想到的思路

  • 第一种方法：
    首先将字符串转换成字符数组，然后提取出所有元音字母的索引，放入一个集合中，然后在字符数组中根据索引集合进行 344 题的解决方法字符串，这种方式需要一个额外的集合存储空间

  • 第二种方法：
    这种方法使用了快速排序中寻找切分点的思想，从字符数组左右两端分别向中间遍历，当遇到元音字符时，交换左右两个指针的元素值，直到左右指针相遇，这种方法不需要额外的空间，性能比第一种方法好很多，只需要一次遍历即可完成

283.Move Zeroes

Description:

Given an array nums, write a function to move all 0’s to the end of it while maintaining the relative order of the non-zero elements.

Example:

given nums = [0, 1, 0, 3, 12], after calling your function, nums should be [1, 3, 12, 0, 0].

Note:

You must do this in-place without making a copy of the array.
Minimize the total number of operations.

解决：
初看到这道题时，可能会想到用排序的思想求解，使用插入排序，将 0 移动到数组最右边，这种方法虽然能够解决问题，但是性能不是很好，甚至有点糟糕，因为它的时间复杂度平均是 O(n^2)，最初我的解决办法就是这个，好尴尬…，下面给出讨论里面看到的一种解决办法：

遍历数组，遇到 0，则计数器加 1，遇到非 0，则将当前索引的值赋值给 (当前索引 - 计数器数值) 的元素

1. Two Sum

Description:

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution.

Example:

Given nums = [2, 7, 11, 15], target = 9,

Because nums[0] + nums[1] = 2 + 7 = 9,

return [0, 1].

解决：

第一种方法：

暴力法，两层循环，遍历数组，找到两个元素的和为 target 的索引，返回。时间复杂度是 O(n^2)

第二种方法：

用一个 HashMap，其中 key 是数组中元素的值，value 是值对应的索引，遍历数组，找到 key 为(target- 数组当前位置的值) 的value，如果存在，则返回当前索引和这个 value，如果不存在，则将当前值作为 key，当前索引作为 value 存入 HashMap 中。时间复杂度是 O(n)
