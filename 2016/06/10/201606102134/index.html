Java 基本数据类型之间转换

09AE9C's Blog
2016-06-10
Java
Java

计算机中数据的基本单位是比特(bit)，或者叫做“位”，一个比特只能表示 0 或 1，在比特之上的单位是字节(Byte)，1 Byte = 8 bit，下表是 java 中各种基本数据类型占用存储空间的大小

数据类型 大小(bit) 最小值 最大值 包装器类
byte 8 -128 127 Byte
boolean 8 - - Boolean
short 16 -2^15 2^15+1 Short
int 32 -2^31 2^31+1 Integer
long 64 -2^63 -2^63+1 Long
float 32 -3.403E38 3.403E38 Float
double 64 -1.798E308 1.798E308 Double

不同数据类型占用的空间不同，能够表示的数据范围也不同，因此要实现不同数据类型之间的转换，需要考虑数据类型能表示的最大最小范围，在 java 中不同数据类型之间转换的规则是：小的数据类型向大的数据类型转换，可以直接转换(自动类型转换)，而大的数据类型向小的数据类型转换，超出部分自动舍弃(强制类型转换)，这也是符合正常逻辑的，就像两个杯子装水一样，一个小杯子，一个大杯子，当小杯子向大杯子倒水时，空的大杯子始终能够装下小杯子中所有的水，而当大杯子向小杯子倒水时，小杯子装满后，大杯子中剩余的水就洒在外面了。

示例

一个 int 类型数据的大小是 32 位(4 字节)，能表示的数字范围是 [-2^31 ~ 2^31+1]，而一个 byte 类型数据的大小是 8 位(1 字节)，能表示的数据范围是 [-128 ~ 127] ，显然 int 的范围 比 byte 大，在 int 数据类型向 byte 数据类型转换的过程中，始终只取 int 数据的最后一个字节，即最后 8 位；而在 byte 数据类型向 int 数据类型转换时,直接将 byte 数据从右往左放入 int 数据的对应位就可以了，因为 int 的容量大于 byte，所以不用考虑有超出的情况，但是会有 3 个字节的空余部分，这个空余部分的数据会根据 byte 的首位来决定是存放 1 还是 0，(即 byte 数据中的符号位)，如果是 1，则空余部分全部填充 1，否则填充 0，这里涉及到原码，补码等机器码相关的知识，可以参考 这篇文章

int 类型转 byte 类型

例如一个 int 类型数据 int a = 443，它的二进制表示是 00000000 00000000 00000001 10111011，转换成 byte 时只取最后的一个字节 10111011，由于 Java 是用补码存储整数的，二进制形式数据中的首位是符号为，0 表示正数，1 表示负数，显然这里的 10111011 是一个负数，它的值需要通过补码转换得到

补码的转换规则是：

从左往右第一位是符号为，0 正 1 负，如果符号位是 0，这个数是正数，补码等于原码，如果符号位是 1，这个数是负数，剩余的数按位取反，再加 1，加上负号

根据补码的转换规则，示例中的 10111011 是一个负数。看剩余的其他位，按照 按位取反，再加 1 进行转换，剩下的 7 个位，每个位取反，然后最后的结果加 1 得到的就是最终的值(如果是负数，还要加上负号)，示例中剩下 7 位分别是 0111011 则按位取反的结果是 1000100,这个值换算成十进制就是 68,然后再加 1，就是 69，最后加上负号，得到的最终结果就是 -69, int 类型数 433 转换成 byte 类型的结果是 -69

byte 类型转 int 类型

接下来再看 byte 类型转 int 类型，假设还是刚才那个数 byte 类型的 -69,如果直接执行 byte b = -69; int i = b;,那 i 的值依然是 -69，并没有变化。原因是：

变量 b 的二进制表示为 10111011,如果直接将它往 变量 i 的存储空间放，结果是: 11111111 11111111 11111111 10111011，这个值就是 -69。

而要转换成正确的 int 类型数，只需要将原 byte 类型数和 0x00FF 执行逻辑与操作。原理就是去掉符号位的干扰，因为一旦 byte 是一个负数，如果直接将它转成 int 类型，则这个 int 类型也便成了负数，然而在 int 数据中，最后一个字节的首位并不是符号位。在这个例子中，将 b 和 0x00FF 执行逻辑与：10111011 & 00000000 11111111,这样存放在 i 中的就是 00000000 00000000 00000000 10111011,也就是 187

以上就是一个简单的基本数据类型转换的例子，主要涉及到二进制代码的转换，更多基本数据类型转换的情况可以参看 java 中的 ByteBuffer 和 Bits 类中的代码实现
