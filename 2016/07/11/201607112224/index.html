Python
Flask, Linux, Python

添加用户

首次登录到 Linux 系统中，只有一个 root 用户，而使用 root 用户来管理应用的话，是存在风险的(误删系统文件等)，一般是建议使用子用户来管理应用，因此首先需要添加用户

接下来按照提示输入密码，输入确认密码，然后后面的步骤一般都使用默认配置即可，添加完成然后退出服务器，就能以用户 tc 进行登录了

配置开发环境

基础的系统环境基本配置好了，如果在实际配置中遇到其他问题，建议在网上搜索下，一般都能找到解决办法的，下面开始配置开发环境

先明确方案，用 Python 来搭建一个小型的后台应用程序，通过在网上查找资料，基本确定使用 Flask + uWsgi + Nginx 的组合

  • Flask: Python 中的一个轻量级 Web 开发框架，简单易用，功能强大，关于它的使用可以参考《Flask Web开发:基于Python的Web应用开发实战》这本书
  • uWsgi: 在生产环境下部署配置管理工具
  • Nginx: 是轻量级、性能强、占用资源少，能很好的处理高并发的反向代理软件

搭建本地同步开发环境

服务器应用开发一般是现在本地开发环境进行开发，然后将测试通过的代码发送到服务器上运行，因此在本地需要有一个与服务器环境一样的模拟开发环境，方便进行开发调试，这里我使用的是 Vagrant 来搭建本地开发环境，不同于其它虚拟机产品，Vagrant 最大的好处就是同步目录，即本地文件目录与 Vagrant 虚拟机中的文件目录是同步的，在本地同步目录中添加文件，虚拟机中也能看到并使用这个文件，非常好用！关于 Vagrant 的使用，可以参考官方文档
