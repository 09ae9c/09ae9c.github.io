配置远端主机

建议在远端主机创建一个新的账号，保持干净

1. 配置 SSH

Mainframer 要求 SSH 免密登录，所以需要把本地的 ~/.ssh/id_rsa.pub 添加到远端的 ~/.ssh/authorized_keys 文件中

做完这个可以实现在本地通过调用 ssh {username}@{remote_host} 命令不需要密码直接登录到远端

2. 必备环境配置，将 Android 编译必备的 SDK，Gradle 配置好

2.1 安装配置 Gradle

更多安装方式可以参考官方文档

2.2 安装配置 Android SDK

官网下载 对应版本的 SDK tools

然后添加环境变量

坑

  1. 目前 Mainframer 的 SSH 配置会有一点问题，SSH 连上远端后拿不到环境变量，所以编译时的环境中就没有 $ANDROID_HOME 这个变量，需要手动在项目的 local.properties 文件中添加 sdk 路径，由于远端的 SDK 路径跟本地不同，所以本地的 ignore 文件中还是要把 local.properties 加入过滤列表，不上传，然后在远端的项目目录下手动创建 local.properties 文件，添加 sdk 路径

  2. 如果开启 Instant Run，会因为远端编译结果的目录和本地安装时定位的目录不是同一个而导致安装失败，所以建议关闭 Instant Run 运行
